    OIZPA API Docs

    Welcome to oizpa's fintech API! You can use our API to integrate with our finance analysis system to help you make decisions for your customers.

    Using an auth token that is availble here, you can do the following:

    • Create an authorization token linked to yours for each CU partner

    • Delete a CU partner authorization token

    • Get a list of systems oizpa can integrate with

    • See callback specs for receiving updates on completed integrations

    Use code snippits here to test and use our API.

    Authentication

    oizpa uses API keys to allow access to the API. You can request an authorization token here.

    oizpa expects an API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: authorization-token

    You must replace authorization-token with your API key (from signing up, or by requesting an individual CU partner token).

    Get a customer API key

    curl -X POST http://developer.oizpa.tech/api/keys -H 'Authorization: authorization-token'

    The above command returns JSON structured like this:

    {
  "id": "some-id",
  "api_key": "auth-token"
}

    This endpoint creates a new auth key for customer use.

    Each CU partner should have have and use their own API key.

    HTTP Request

    POST http://developer.oizpa.com/keys

    Request Headers

    Parameter Required Description
    authorization Yes The secret key issued to you.

    Delete an API key

    curl -X DELETE http://developer.oizpa.tech/api/<ID> -H 'Authorization: authorization-token'

    The above command returns JSON structured like this:

    {
  "id": "key-id-deleted"
}

    This endpoint deletes a specific API key.

    HTTP Request

    DELETE http://developer.oizpa.com/keys/<ID>

    Request Headers

    Parameter Required Description
    authorization Yes The authorization tokens issued to you.

    Integrations

    To get a list of possible integration methods

    curl -X GET http://developer.oizpa.tech/api/integrations -H 'Authorization: authorization-token'

    The above command returns JSON structured like this:

    [
  {
    "type": "Download",
    "imageUrl": "http://developer.oizpa.tech/quickbooksDesktop.png",
    "qwcUrl": "http://developer.oizpa.tech/qwc",
    "instructionsUrl": "http://developer.oizpa.tech/qbdInstructions.pdf"
  },
  {
    "type": "cloud",
    "imageUrl": "http://developer.oizpa.tech/quickbooksOnline.png",
    "cloudUrl": "http://developer.oizpa.tech/oAuth",
    "instructionsUrl": "http://developer.oizpa.tech/qbdInstructions.pdf"
  }
]

    This endpoint retrieves all integration options.

    HTTP Request

    GET http://developer.oizpa.tech/api/integrations

    Request Parameters

    Parameter Required Description
    authorization Yes Required parameter to use ozipa APIs
    customer_id No If set, links the requested integration to a specific ID associated with your authorization key.
    That's it!

    Troubleshooting

    This section helps with common issues you may run into. If this section does not answer your question, please use the Intercom widget [here](http://developer.oizpa.tech)

    The oizpa API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is missing something, or has bad data in it.
    401 Unauthorized -- Your authorization token is wrong. Did you sign-up for one?
    404 Not Found -- The specified endpoint doesn't exist.
    405 Method Not Allowed -- You tried to make an API call with the wrong method (GET/POST/DELETE)
    500 Internal Server Error -- It's not you, it's me
    503 Service Unavailable -- It's not you, it's me
